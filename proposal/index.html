Overview

A brief description of the project.

The project will be Chess against the Stockfish chess engine.

It will also feature board analysis from Lichess.

What is the problem you are trying to solve?

There is not really an "problem" I am solving. It's just Chess against a robot.

What is your motivation to work on this?

I have been a fan of Chess for a long time, (though I am not particularly good at it).

For every final project I have had thus far, I wished I had been able to do Chess, and this time I believe that I can pull it off.

Target Audience

Who is the target audience for the application? Who will it serve?

It is targeted at people that want to play Chess, and more specifically people that want to get better at it.

Major Functions

setupBoard()
Builds the game board and places all pieces in their proper starting positions.
getAiMove()
Fetches the API response on where the opponent should move. Sends the current board state using FEN notation.
getBoardAnalysis()
Fetches board analysis API response from Lichess.
buildFenString()
Takes the current board state and converts it into a valid FEN string to send to the APIs.
clickSpace()
Called when a space is clicked. Gathers information to send to other functions, such as: which space was clicked, what piece is in it, what color the piece is, and whether the piece has moved (important for pawns, rooks, and kings).
showMoves()
Highlights valid spaces to move to, including when kings are in check, and does nothing if there are no valid moves from said space.
[pawn|knight|bishop|queen|king]Move()
Called by showMoves. Gets valid moves for the piece type given the current board state.
makeMove()
Actually makes the move happen. Updates where the selected piece is located and captures pieces as needed.
updateHistory()
Adds new lines to the move history display by calling buildFenString.

Chess engine for opponent

Board analysis for player

What data will you need to store?

I believe it would be reasonable to store the current game in progress in localStorage. Chess games can go on for a while, and it would be most unfortunate if someone were to loose power and need to start over.

Module List

  • Game.mjs - for general game logic, such as keeping track of turns and updating the history
  • Board.mjs - for board based logic such as where everything is and what piece is what
  • Piece.mjs - would be a piece class holding all the relevant data for said piece
  • Move.mjs - for determining valid moves
  • API.mjs - for interacting with the APIs
